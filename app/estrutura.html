Explicação da Estrutura

⚙️

.github/workflows

Contém o ficheiro YAML do GitHub Actions que define a pipeline CI/CD.

🖥️

app/

Contém todos os ficheiros da aplicação web: páginas HTML, CSS, JavaScript e imagens.

🐳

Dockerfile

Define como a imagem Docker é construída para servir a aplicação com Nginx.

📘

README.md

Documenta o projeto, a pipeline, a convenção de commits e a execução local.

Como os Componentes se Ligam

Aplicação

O conteúdo da pasta app/ representa a interface do projeto e é o que será servido ao utilizador final.

Docker

O Dockerfile copia o conteúdo da pasta app/ para dentro do container baseado em Nginx.

GitHub Actions

O workflow automatiza a validação dos commits e executa o build da imagem Docker.

GHCR

A imagem Docker construída pela pipeline é publicada no GitHub Container Registry.

Vantagem desta organização

Esta estrutura torna o projeto claro, modular e fácil de apresentar. A aplicação fica separada da automação e da documentação, o que melhora manutenção, leitura e escalabilidade.
