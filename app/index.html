Pipeline CI/CD com Docker e GitHub Actions

Este projeto demonstra a automatização do processo de validação, construção e publicação de uma aplicação web estática usando Docker, GitHub Actions e GitHub Container Registry.

Ver Workflow Ver Docker

Sobre o Projeto

O objetivo deste trabalho é demonstrar a criação de uma pipeline CI/CD para uma aplicação web simples. A aplicação base é um “Hello World” estático, mas o foco principal está na automação do processo através do GitHub Actions e na utilização do Docker para empacotamento.

Sempre que é feito um push para o repositório, a pipeline verifica se a mensagem de commit segue a convenção definida, constrói uma imagem Docker e publica automaticamente essa imagem no GitHub Container Registry.

Conceitos Fundamentais

Automação

Permite reduzir tarefas manuais e garantir um processo consistente desde o commit até à publicação da imagem.

Continuous Integration

Valida automaticamente alterações no código sempre que estas são enviadas para o repositório.

Continuous Delivery

Prepara e publica automaticamente o artefacto final, neste caso uma imagem Docker.

📦

Container

Um container é um ambiente isolado onde a aplicação corre com tudo o que precisa para funcionar.

Fluxo Geral

1

Push

O utilizador envia alterações para o repositório.

2

Validação

A pipeline valida a mensagem do commit.

3

Build

A imagem Docker é construída a partir do projeto.

4

Publicação

A imagem é publicada no GitHub Container Registry.

Tecnologias Utilizadas

Docker

Utilizado para empacotar a aplicação e garantir portabilidade entre ambientes.

GitHub Actions

Responsável pela automação da pipeline CI/CD com base em eventos do repositório.

GHCR

Serviço utilizado para armazenar e distribuir a imagem Docker gerada pela pipeline.
