Docker e Containerização

Explicação do papel do Docker, do Dockerfile e da utilização do Nginx neste projeto.

O que é Docker

Docker é uma plataforma que permite criar, empacotar e executar aplicações dentro de containers. Um container inclui a aplicação e os recursos necessários para a sua execução, garantindo que tudo corre da mesma forma em ambientes diferentes.

Neste projeto, Docker é usado para empacotar a aplicação web estática e permitir que ela seja distribuída de forma simples, consistente e reprodutível.

Conceitos Fundamentais

📦

Imagem

É o artefacto criado a partir do Dockerfile. Serve como modelo para executar containers.

📁

Container

É a instância em execução de uma imagem Docker.

📝

Dockerfile

Ficheiro que descreve passo a passo como construir a imagem Docker.

🌐

Nginx

Servidor web usado para disponibilizar os ficheiros estáticos da aplicação.

Dockerfile Utilizado

FROM nginx:alpine

COPY app/ /usr/share/nginx/html/

Explicação Linha por Linha

FROM nginx:alpine

Define a imagem base do projeto. Neste caso, é utilizada a imagem oficial do Nginx numa versão leve baseada em Alpine Linux.

COPY app/ /usr/share/nginx/html/

Copia o conteúdo da pasta app/ para a diretoria padrão do Nginx, onde os ficheiros web ficam disponíveis para serem servidos.

Porque esta abordagem foi usada?

Como a aplicação é estática, não existe necessidade de um servidor complexo ou de um backend dedicado. O Nginx é uma escolha leve, conhecida e apropriada para servir ficheiros HTML, CSS, JavaScript e imagens.

A imagem nginx:alpine também é adequada porque é pequena e rápida, o que torna o build mais eficiente e facilita a publicação no registry.

Execução Local

docker build -t hello-world .

docker run -p 8080:80 hello-world

O primeiro comando constrói a imagem Docker. O segundo arranca um container a partir dessa imagem e expõe a aplicação localmente na porta 8080.

Depois disso, a aplicação pode ser acedida no browser através de http://localhost:8080.
